## 1) Manifest stays a dict
**Approve.**

One suggestion: use **small helper accessors** for common nested paths instead of raw indexing everywhere. Not full dataclasses — just helpers like:

**Approve, with one caveat.**

I agree the manifest is not an LLM-facing boundary, so it doesn’t need the same validator posture as:
- `last_scan.json`
- `compile_result.json`

But I **would** add a tiny internal invariant check function, something like:

- `assert_manifest_invariants(manifest)`

Run it:
- after `apply_scan_reconciliation`
- after `apply_compile_result`
- before write

Not a separate tool/module necessarily — but definitely a reusable function, not scattered inline assertions only.

Why:
- catches internal corruption early
- easier to debug than ad hoc asserts
- keeps “no manifest validator” while still enforcing sanity

So: no standalone `validate_manifest.py`, yes to **internal invariant checker**.

# Two implementation notes I’d insist on

## 1) Journal-then-pointer write order
You already called this out with D15 — good.

Write order should be:

1. build `next_manifest`
2. build `journal`
3. atomically write `runs/<run_id>.json`
4. atomically replace `manifest.json`

If step 3 fails: stop  
If step 4 fails: journal exists, manifest pointer not advanced — that’s acceptable and recoverable

That’s the right failure mode.

## 2) `incoming_links_known` should be fully derived
Do not try to incrementally patch it.  
Recompute it from all `outgoing_links` every run.

That is:
- simpler
- deterministic
- less bug-prone

At your scale, recomputation is cheap.

My recommendation:

- patch applier / file writer can separately track actual file writes if needed

Otherwise “touched” gets ambiguous.

# Overall module shape

`manifest_update.py` should have three layers:

1. **Load / write shell**
   - read JSON inputs
   - call pure transformation pipeline
   - atomically write outputs

2. **Pure transform core**
   - reconcile scan results
   - apply compiler result
   - recompute derived fields
   - build journal

3. **Internal invariant checks**
   - cheap sanity validation before write

# Proposed top-level flow

```text name=manifest-update-flow.txt
load prior manifest
load last_scan
load compile_result
build run_ctx

next_manifest = deep copy of prior manifest
next_manifest = apply_scan_reconciliation(next_manifest, last_scan, run_ctx)
next_manifest = apply_compile_result(next_manifest, compile_result, run_ctx)
next_manifest = reconcile_incoming_links(next_manifest)
next_manifest = recompute_stats(next_manifest)
assert_manifest_invariants(next_manifest)

journal = build_journal(prior_manifest, next_manifest, last_scan, compile_result, run_ctx)

if write:
  write_outputs(next_manifest, journal, state_root)

return next_manifest, journal
```

## 3) `apply_scan_reconciliation(manifest, last_scan, run_ctx)`

### Purpose
Apply authoritative filesystem delta from scanner:
- `MOVED`
- `DELETED`

This is the step that reconciles source disappearance/renames before compile payload is applied.

### Signature
```python
def apply_scan_reconciliation(manifest: dict, last_scan: dict, run_ctx: dict) -> dict:
```

### Inputs used
- `last_scan["to_reconcile"]`
- optionally `last_scan["files"]` for fresh metadata on moved target

### Responsibilities

#### A. Handle `MOVED`
For each:
```json
{ "type": "MOVED", "from": old_path, "to": new_path }
```

Do:
- move active source record `sources[old_path]` → `sources[new_path]`
- update:
  - `source_id`
  - `canonical_path`
  - `status = active`
  - `compile_state = moved_source`
  - `last_seen_at`
  - `last_run_id`
- add tombstone at `tombstones[old_path]`
- update all `pages[*].source_refs[*].source_id`
- update all `pages[*].supports_page_existence`

#### B. Handle `DELETED`
For each:
```json
{ "type": "DELETED", "path": source_id }
```

Do:
- add tombstone
- remove active source entry
- remove source from all:
  - `source_refs`
  - `supports_page_existence`
- if a page loses all `supports_page_existence`:
  - mark `status = orphan_candidate`
  - `orphan_candidate = true`
  - add `manifest["orphans"][page_id]`

### Invariants
- no active source remains at old moved path
- deleted source not left in `supports_page_existence`
- orphan registry matches orphan page flags

### Tests
- moved source updates page refs correctly
- deleted source marks unsupported page orphan
- deleted source with remaining supports does not orphan page
- tombstones recorded
- `previous_versions` not expanded here except optional move note

---

## 4) `apply_compile_result(manifest, compile_result, run_ctx)`

### Purpose
Apply compiler-authored source/page updates to manifest.

### Signature
```python
def apply_compile_result(manifest: dict, compile_result: dict, run_ctx: dict) -> dict:
```

### Responsibilities

#### A. Upsert source records
For each `compiled_source`:
- upsert `manifest["sources"][source_id]`
- set/update:
  - `hash`
  - `mtime`
  - `size_bytes`
  - `file_type`
  - `status = active`
  - `last_seen_at`
  - `last_compiled_at`
  - `last_run_id`
  - `compile_state`
  - `compile_count`
  - `summary_page`
  - `outputs_created`
  - `outputs_touched`
  - `concept_ids`
  - `link_operations`
  - `provenance`

#### B. Push `previous_versions`
If source existed before and `old_hash != new_hash`:
- append previous version snapshot
- cap list to last 20

#### C. Upsert page records
For each `compiled_source.pages[]`:
- create or update `manifest["pages"][page_id]`
- preserve `created_at` if page existed
- set/update:
  - `page_type`
  - `status`
  - `title`
  - `concept_id`
  - `updated_at`
  - `last_run_id`
  - `source_refs`
  - `supports_page_existence`
  - `outgoing_links`
  - `incoming_links_known` (temporarily accept; later recomputed globally)
  - `confidence`
  - `orphan_candidate`
  - `last_link_reconciled_at`

#### D. Clear resolved orphan flags
If a page is now active and supported again:
- remove from `manifest["orphans"]`

#### E. Handle scan-listed compile omissions
If a path is in `last_scan["to_compile"]` but missing from `compile_result["compiled_sources"]`:
- decide policy

### Recommended policy
Treat as compiler failure / error-state source record:
- `status = error`
- `compile_state = error`

Do **not** silently ignore.

### Invariants
- every compiled source has an active manifest source record
- every page in compile result has a page record
- created pages include current source support

### Tests
- new source inserted correctly
- changed source pushes previous_versions
- previous_versions caps at 20
- orphan page reactivated correctly
- omitted to_compile source marked error

## 5) `reconcile_incoming_links(manifest)`

### Purpose
Derive `incoming_links_known` from everyone’s `outgoing_links`.

### Signature
```python
def reconcile_incoming_links(manifest: dict) -> dict:
```

### Behavior
- ignore any previously stored `incoming_links_known`
- recompute from scratch

Algorithm:
1. initialize empty inbound map for all pages
2. for each page A:
   - for each `target` in `A.outgoing_links`:
     - if target exists in manifest pages:
       - append A to inbound[target]
3. write sorted unique inbound lists back

### Why full recompute
- deterministic
- avoids stale backlinks
- cheap at current scale

### Tests
- backlinks computed correctly
- stale incoming links disappear
- self-links preserved or omitted depending on policy

### Recommendation
You may keep self-links out here even if they slip into outgoing data.

## 6) `recompute_stats(manifest)`

### Purpose
Refresh manifest counters and top-level timestamps.

### Signature
```python
def recompute_stats(manifest: dict, run_ctx: dict) -> dict:
```

### Updates
- `stats.total_raw_files = len(sources)`
- `stats.total_pages = len(pages)`
- page type counts
- `stats.total_runs += 1`
- `runs.last_run_id = run_ctx["run_id"]`
- if success:
  - `runs.last_successful_run_id = run_ctx["run_id"]`
- `updated_at = run_ctx["finished_at"]`

### One important note
Be careful with `total_runs += 1`:
- only do it once per successful invocation
- if rerunning idempotently with same `run_id`, you may double-count

### My recommendation
If `runs.last_run_id == run_ctx["run_id"]`, do **not** increment.

### Tests
- counts correct
- type breakdown correct
- rerun with same run_id doesn’t double-increment

## 7) `assert_manifest_invariants(manifest)`

### Purpose
Internal sanity checker before write.

### Signature
```python
def assert_manifest_invariants(manifest: dict) -> None:
```

### Suggested checks
- all top-level keys exist:
  - `sources`, `pages`, `orphans`, `tombstones`, `runs`, `stats`
- every orphan entry has corresponding page
- every orphan page has:
  - `status = orphan_candidate` or `stale`
  - `orphan_candidate = true`
- every page’s `supports_page_existence` sources either:
  - exist in active sources
  - or you intentionally allow tombstoned historical refs
- no duplicate page IDs or source IDs
- `incoming_links_known` contains only known page IDs
- `previous_versions` length <= 20 for every source

### Recommendation
Raise `RuntimeError` or `AssertionError` with specific messages.

### Tests
- malformed orphan state fails
- overlong previous_versions fails
- dangling incoming link fails
